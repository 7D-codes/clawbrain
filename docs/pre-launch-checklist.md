# ClawBrain Pre-Launch Checklist

Complete everything marked **REQUIRED** before launch. **NICE TO HAVE** can slip.

---

## TECHNICAL (REQUIRED)

### Product Stability
- [ ] Core workflow works without crashes (chat → task → board)
- [ ] No data loss bugs
- [ ] Handles edge cases gracefully
- [ ] Tested on multiple browsers (Chrome, Firefox, Safari)
- [ ] Mobile web is usable (not perfect, but functional)
- [ ] Onboarding flow is clear (new user can get started in 5 min)

### Performance
- [ ] Page load under 3 seconds
- [ ] Chat responses feel snappy
- [ ] Kanban board handles 100+ tasks smoothly
- [ ] No memory leaks in long sessions

### Security
- [ ] No exposed API keys in client
- [ ] Basic input sanitization
- [ ] Rate limiting on API endpoints
- [ ] Secure WebSocket connections

### Data
- [ ] Import works (from common formats)
- [ ] Export works (markdown files)
- [ ] Backup/restore documented
- [ ] Data migration path for updates

---

## MARKETING (REQUIRED)

### Landing Page
- [ ] Live at clawbrain.io (or domain)
- [ ] Clear headline and subhead
- [ ] Demo video or GIF (shows core workflow)
- [ ] Email capture working
- [ ] Privacy policy page
- [ ] Terms of service page
- [ ] Mobile responsive
- [ ] Loads fast (<3 sec)

### Email
- [ ] Email provider set up (ConvertKit/Buttondown)
- [ ] Welcome email automated
- [ ] Sender domain authenticated (SPF/DKIM)
- [ ] Unsubscribe link in all emails

### Social
- [ ] Twitter/X profile optimized (bio, pinned tweet)
- [ ] LinkedIn profile optimized
- [ ] Discord server active (10+ members)
- [ ] GitHub repo polished (README, screenshots)

### Content
- [ ] 3+ blog posts published
- [ ] Content calendar for launch week
- [ ] Email sequence for launch (announcement, follow-up)
- [ ] Social posts scheduled for launch day

## PRE-LAUNCH TESTING

### User Testing
- [ ] 3+ fresh users onboarded without help
- [ ] Watch them use the product (screen share)
- [ ] Note confusion points
- [ ] Fix critical issues

### Launch Day Simulation
- [ ] Test signup flow 10 times
- [ ] Test email delivery
- [ ] Test all CTAs
- [ ] Check all links
- [ ] Review on mobile

### Stress Testing
- [ ] Can server handle 100 concurrent users?
- [ ] Can it handle 1,000? (if going viral)
- [ ] Database performance under load

---

## LAUNCH DAY CHECKLIST

### Morning (6 AM - 12 PM)
- [ ] Final check of all systems
- [ ] Product Hunt goes live
- [ ] Email announcement sent
- [ ] Twitter thread posted
- [ ] LinkedIn post published
- [ ] Hacker News "Show HN" posted
- [ ] Discord announcement

### Afternoon (12 PM - 6 PM)
- [ ] Reply to every Product Hunt comment
- [ ] Monitor Twitter, engage with mentions
- [ ] Check email for issues
- [ ] Monitor server performance
- [ ] Post updates on progress

### Evening (6 PM - 12 AM)
- [ ] Thank supporters publicly
- [ ] Share early results
- [ ] Monitor for critical bugs
- [ ] Prepare day 2 content
- [ ] Rest (important!)

### Throughout Day
- [ ] Screenshot key moments (ranking, comments)
- [ ] Save testimonials as they come in
- [ ] Track metrics hourly
- [ ] Stay calm and responsive
